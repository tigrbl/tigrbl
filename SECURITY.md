# Security Policy

## Supported lines

This repository is currently in a pre-certification checkpoint state. No formal long-term support matrix is declared yet. Security fixes should target the active development line represented by the current repository state.

## Reporting a vulnerability

Please do **not** open a public issue for a suspected vulnerability.

Use the repository hosting platform’s private vulnerability reporting or security advisory workflow to report:

- authentication or authorization flaws
- OpenAPI / docs-surface exposure issues
- request handling, middleware, or runtime vulnerabilities
- engine/session/transaction vulnerabilities
- packaging or supply-chain concerns

## What to include

Provide:

- affected package(s) or path(s)
- a concise description of impact
- reproduction steps or proof of concept when safe to share privately
- any environment assumptions
- suggested mitigations if available

## Response expectations

This Phase 0 checkpoint establishes the policy surface only. Formal SLAs, supported release windows, and external disclosure rules should be tightened during the later certification phases.
